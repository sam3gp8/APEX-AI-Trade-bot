Upload the APEX add-on files to your GitHub repository. Two methods — choose whichever you prefer.

Method A — GitHub Web Upload (Easiest)

Upload files directly through github.com — no Git required.

Add file Upload files ← Click this
  1. Go to your repo on GitHub: click here to open →
  2. Click Add file → Upload files
  3. Drag the entire apex-trading-addon/ folder into the upload box
  4. Scroll down, write commit message: feat: initial APEX add-on release
  5. Click Commit changes
⚠️
GitHub's web uploader has a 100 file / 25MB limit. The APEX zip is well within this. If you have issues, use Method B below.

Method B — Git Command Line

For users comfortable with Git. Requires Git installed.

bash — run on your computer
# 1. Clone your new empty repo
git clone https://github.com/YOUR_USERNAME/apex-trading-addon.git
cd apex-trading-addon

# 2. Extract the APEX zip and copy files here
#    (run from the directory where apex-trading-addon.zip is)
unzip ../apex-trading-addon-v2.zip -d /tmp/apex-src
cp -r /tmp/apex-src/apex-trading-addon/* .

# 3. Stage and commit everything
git add -A
git commit -m "feat: initial APEX Trading AI add-on release v1.0.0"

# 4. Push to GitHub
git push origin main

04
Update Your Details
A few files reference YOUR_GITHUB_USERNAME — update them with your real details.
💡
You can edit files directly on GitHub — click the file, then the pencil ✏️ icon. Or use the git command line if you prefer.

Files to Update

Replace YOUR_GITHUB_USERNAME in these three files with your username:

Files needing your username # 1. apex_trading/config.yaml — line 5: url: "https://github.com/YOUR_USERNAME/apex-trading-addon" → change to: url: "https://github.com/YOUR_USERNAME/apex-trading-addon" # 2. repository.json: "url": "https://github.com/YOUR_USERNAME/apex-trading-addon" → change to: "url": "https://github.com/YOUR_USERNAME/apex-trading-addon" # 3. README.md — replace all occurrences of YOUR_GITHUB_USERNAME sed -i 's/YOUR_GITHUB_USERNAME/YOUR_USERNAME/g' README.md

Or use this single command from your cloned repo directory:

bash
# Replace YOUR_GITHUB_USERNAME with your actual username in all files
find . -type f \( -name "*.yaml" -o -name "*.json" -o -name "*.md" -o -name "*.sh" \) \
  -exec sed -i 's/YOUR_GITHUB_USERNAME/YOUR_USERNAME/g' {} +

git add -A
git commit -m "chore: update repository URLs with actual username"
git push
05
Create First Release
Create a GitHub release with the v1.0.0 tag. This triggers the GitHub Actions workflow to build Docker images for all architectures and publish them to the GitHub Container Registry.
ℹ️
GitHub Actions builds automatically when you push a tag starting with v. The first build takes ~10-15 minutes (building for 5 CPU architectures). Subsequent builds use cache and are much faster.

Option A — Create Release via GitHub UI (Recommended)

  1. Go to your repo: click here →
  2. Click Releases (right sidebar) → Create a new release
  3. Click Choose a tag → type v1.0.0 → + Create new tag: v1.0.0
  4. Title: APEX Trading AI v1.0.0
  5. Description: Initial release — AI trading bot for Home Assistant OS
  6. Click Publish release
  7. Go to the Actions tab to watch the build progress

Option B — Create Tag via Git

bash
# Create and push the v1.0.0 release tag
git tag -a v1.0.0 -m "APEX Trading AI v1.0.0 — Initial release"
git push origin v1.0.0

# Then create the release on GitHub to attach release notes:
https://github.com/YOUR_USERNAME/apex-trading-addon/releases/new?tag=v1.0.0
✅
After pushing the tag, go to Actions tab in your repo. You'll see the "Build and Publish" workflow running. Green checkmarks = success. The Docker images are published to ghcr.io/YOUR_USERNAME/apex-trading-*.
