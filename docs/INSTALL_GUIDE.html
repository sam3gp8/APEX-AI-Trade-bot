OVERVIEW
Why run APEX as a HAOS add-on?
Without HAOS Add-on (Browser-only): Browser → Broker API ✗ CORS BLOCKED Browser → Yahoo Finance ✗ CORS BLOCKED localStorage ✗ Clears on browser wipe Auto-analysis ✗ Only works while tab is open With HAOS Add-on: Python Server → Broker API ✓ Server-side, direct, no CORS Python Server → Yahoo/Finnhub ✓ Real prices, server-side /data/apex/*.json ✓ Permanent, survives reboots Background task loop ✓ Runs 24/7 without browser
✅
Running APEX as a HAOS add-on means your broker API calls happen server-side inside your HA instance — the same network as your home, bypassing all browser CORS restrictions. Market data is fetched by Python on a real server, and your trade history and portfolio data are stored permanently in /data/apex/.

METHOD 1 · SELF-INSTALLER
Automated script — recommended for most users
ℹ️
This method uses the install.sh script included in the APEX zip. It detects your HA environment, downloads all files, sets up data directories, and reloads the add-on registry automatically.
1
Download APEX and Extract the ZIP
Download apex-trading-addon.zip from Claude. Extract it — you'll see a folder called apex-trading-addon/ containing install.sh and the apex_trading/ add-on folder.
2
Open a Terminal on Your HA Host
Option A — HA Terminal add-on (recommended): In HA go to Settings → Add-ons → Terminal & SSH → Open Web UI.

Option B — SSH from your computer:
bash
# Replace with your HA IP address
ssh root@192.168.1.100
# Default HAOS password is set in Settings → System → Security
3
Upload the ZIP to Your HA Host
From your local computer, copy the zip to HA:
bash — run on your computer
# Using SCP (Mac/Linux/Windows with OpenSSH)
scp apex-trading-addon.zip root@192.168.1.100:/tmp/

# Using rsync (faster for repeated transfers)
rsync -avz apex-trading-addon/ root@192.168.1.100:/tmp/apex-trading-addon/
If using the HA Terminal add-on, use the Samba method (Method 4) to upload, then run from there.
4
Run the Self-Installer
On your HA host terminal:
bash — on HA host
# Extract the zip
cd /tmp
unzip apex-trading-addon.zip
cd apex-trading-addon

# Run the installer
chmod +x install.sh
./install.sh
✅
The installer will: detect your environment → copy files to /addons/apex_trading/ → set up data directories → reload the add-on registry → print next steps.
5
Find APEX in the Add-on Store
After the installer completes:
http://homeassistant.local:8123/hassio/store
🏠
🗺️
⚙️
📊
Settings → Add-ons → Add-on Store

AI-powered trading bot with Claude market analysis, live price feeds, portfolio tracking...
⚠️
If you don't see APEX: click the ⋮ menu (top right of Add-on Store) → Check for updates, then hard-refresh the page (Ctrl+Shift+R).

METHOD 2 · TERMINAL ADD-ON
Install Terminal & SSH add-on first, then use it to install APEX
1
Install the Terminal & SSH Add-on
In Home Assistant: Settings → Add-ons → Add-on Store → search "Terminal" → Install Terminal & SSH (official add-on).

After installing: enable Show in sidebar and click Start.
2
Upload APEX Files via Samba
While the Terminal add-on is running, also install the Samba share add-on so you can upload files from your computer. See Method 4 for Samba setup.

Upload the extracted apex-trading-addon/ folder to the addons share.
3
Run Installer from Terminal
Open the Terminal add-on from the HA sidebar, then:
bash — HA Terminal
# The Samba 'addons' share maps to /addons/ on the host
cd /addons/apex-trading-addon
chmod +x install.sh
./install.sh
4
Alternative: Copy Files Directly
If Samba isn't set up, you can install directly using the terminal:
bash — HA Terminal (manual)
# Create addon directory
mkdir -p /addons/apex_trading
mkdir -p /addons/apex_trading/rootfs/opt/apex/static
mkdir -p /addons/apex_trading/rootfs/etc/s6-overlay/s6-rc.d/apex
mkdir -p /addons/apex_trading/rootfs/etc/s6-overlay/s6-rc.d/init-apex
mkdir -p /addons/apex_trading/translations

# Download files directly from GitHub
BASE="https://raw.githubusercontent.com/your-repo/apex-trading-addon/main/apex_trading"
curl -fsSL "${BASE}/config.yaml" -o /addons/apex_trading/config.yaml
curl -fsSL "${BASE}/Dockerfile" -o /addons/apex_trading/Dockerfile
curl -fsSL "${BASE}/rootfs/opt/apex/server.py" -o /addons/apex_trading/rootfs/opt/apex/server.py
curl -fsSL "${BASE}/rootfs/opt/apex/static/index.html" -o /addons/apex_trading/rootfs/opt/apex/static/index.html

# Download s6 service files
curl -fsSL "${BASE}/rootfs/etc/s6-overlay/s6-rc.d/apex/run" \
  -o /addons/apex_trading/rootfs/etc/s6-overlay/s6-rc.d/apex/run
echo "longrun" > /addons/apex_trading/rootfs/etc/s6-overlay/s6-rc.d/apex/type
chmod +x /addons/apex_trading/rootfs/etc/s6-overlay/s6-rc.d/apex/run

# Reload addon registry
ha addons reload

METHOD 3 · SSH + SCP
Traditional SSH access from your computer
⚠️
HAOS requires SSH to be enabled first. Go to Settings → System → Security → enable Advanced mode (user profile), then install the SSH & Web Terminal add-on from the store.
1
Enable SSH on HAOS
1. Click your username (bottom left) → Enable Advanced Mode
2. Settings → Add-ons → Store → search "SSH" → Install Terminal & SSH
3. In the SSH add-on Configuration tab: set a password or add your public key
4. Start the add-on
2
Copy APEX Files to HA
bash — your computer
# Extract the zip first, then SCP the folder
unzip apex-trading-addon.zip

# Copy to HA (replace IP with your HA address)
scp -r apex-trading-addon/apex_trading root@192.168.1.100:/addons/apex_trading
scp apex-trading-addon/install.sh root@192.168.1.100:/tmp/install.sh

# Windows users — use WinSCP GUI or PowerShell:
# scp -r apex-trading-addon\apex_trading root@192.168.1.100:/addons/apex_trading
3
SSH In and Complete Installation
bash — your computer
ssh root@192.168.1.100

# Once connected to HA host:
chmod +x /addons/apex_trading/rootfs/etc/s6-overlay/s6-rc.d/apex/run
chmod +x /addons/apex_trading/rootfs/etc/s6-overlay/s6-rc.d/init-apex/run

# Create data directory
mkdir -p /addon_configs/apex_trading/apex

# Reload addon registry so HA sees the new addon
ha addons reload
# OR restart Home Assistant:
ha core restart

METHOD 4 · FILE SHARE (SAMBA)
Drag & drop via Windows Explorer or macOS Finder — no terminal required
1
Install the Samba Add-on
Settings → Add-ons → Store → search "Samba" → Install Samba share → Start.

Configure a username and password in the add-on Configuration tab.
2
Map the Network Drive
Windows: Open Explorer → This PC → Map network drive → enter \\192.168.1.100\addons (use your HA IP)

macOS: Finder → Go → Connect to Server → enter smb://192.168.1.100/addons

Linux: sudo mount -t cifs //192.168.1.100/addons /mnt/ha-addons -o username=homeassistant
3
Copy the Add-on Folder
Extract the APEX zip. Copy the entire apex_trading folder into the mapped addons network drive. The result should be:
target structure
addons\                          ← the mapped network drive
└── apex_trading\
    ├── config.yaml
    ├── Dockerfile
    ├── translations\
    │   └── en.json
    └── rootfs\
        ├── etc\s6-overlay\...
        └── opt\apex\
            ├── server.py
            └── static\
                └── index.html
4
Reload Add-ons in HA
In Home Assistant UI: Settings → Add-ons → Store → click ⋮ (top right) → Check for updates. APEX will appear under "Local add-ons". Click Install.
💡
If APEX doesn't appear after reloading, try a full HA restart: Settings → System → Restart Home Assistant.

CONFIGURATION
Settings in the Add-on Configuration tab

After installing APEX, go to Settings → Add-ons → APEX Trading AI → Configuration. All settings can also be changed from within the dashboard UI at runtime.

anthropic_api_key REQUIRED
Your Claude AI key from console.anthropic.com. Powers all market analysis, trade signals, and AI chat. Starts with sk-ant-
market_data_source optional
yahoo — free, no key needed (default)
finnhub — free key at finnhub.io
alphavantage — free at alphavantage.co
sim — offline simulation
trading_mode optional
paper — simulation using real prices (default, safe)
live — real order execution via broker API
broker_api_url optional
Your broker's base API URL. Examples:
https://api.alpaca.markets/v2
https://api.binance.com/api/v3
https://api.kraken.com/0
broker_api_key + broker_api_secret optional
Your broker credentials. Stored securely in the HA secrets store. Used for server-side balance sync and order execution.
auto_analysis_interval optional
Seconds between automatic Claude market scans.
0 = manual only
600 = every 10 min (default)
3600 = every hour
stop_loss_pct / take_profit_pct
Default risk parameters per trade. 2.0 = 2% stop loss. 6.0 = 6% take profit. Can be overridden per trade in the UI.
tax_bracket
Your income tax rate as a decimal. 0.24 = 24%. Used for tax optimization calculations and LTCG recommendations.
config.yaml — minimum recommended setup
anthropic_api_key: "sk-ant-YOUR-KEY-HERE"
market_data_source: "yahoo"      # free, no key needed
trading_mode: "paper"            # safe simulation to start
risk_level: "moderate"
stop_loss_pct: 2.0
take_profit_pct: 6.0
max_position_pct: 5.0
auto_analysis_interval: 600    # Claude scans every 10 min
log_level: "info"

Getting APEX up and running after installation
1
Start the Add-on
In HA: Settings → Add-ons → APEX Trading AI → click Start. Watch the Log tab for startup messages. You should see:

APEX Trading AI starting up... | Market data source: yahoo | Server starting on port 7123
2
Open the Dashboard
Three ways to access APEX:
access URLs
# Option 1: HA sidebar (after enabling ingress)
Click "APEX Trading" in the HA left sidebar

# Option 2: Direct URL
http://YOUR-HA-IP:7123

# Option 3: HA ingress URL
http://homeassistant.local:8123/hassio/ingress/apex_trading
3
Set Your Starting Capital
In the APEX sidebar, find the 💵 Account Capital panel. Enter your brokerage account balance and click Set Starting Capital. This is your baseline for all P&L calculations.

If your broker supports direct API balance fetching (Alpaca, Binance, etc.), connect your broker first for automatic sync.
4
Fetch Live Prices
In the 📡 Market Data panel, click Fetch Live Prices Now. APEX will pull real prices server-side via Yahoo Finance (no key needed). You'll see all 10 symbols update in the ticker strip.

Run Your First Claude Analysis
Go to the 🤖 Claude Analysis tab and click 🔍 Full Market Scan. Claude will analyze all monitored symbols and generate BUY/SELL/HOLD signals with specific entry/target/stop prices. The analysis is saved to /data/apex/analyses.json permanently.
💡
Set auto_analysis_interval: 600 in config to have Claude scan automatically every 10 minutes while the bot is active — no browser required.
6
Activate the Bot
Click the BOT toggle in the header. The bot will:
📡 Fetch real prices every 30 seconds
🧠 Auto-optimize strategy weights after each paper trade
📊 Size positions based on your actual capital and risk config
💰 Track all P&L, wins/losses, and tax implications
🤖 Run Claude market scans on your configured schedule

TROUBLESHOOTING
Common issues and how to fix them
APEX doesn't appear in the Add-on Store after installation
1. Go to Settings → Add-ons → Store → click ⋮ → Check for updates
2. Hard refresh the HA page: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac)
3. Check the add-on folder exists: ls /addons/apex_trading/config.yaml — if this fails, the files weren't copied correctly
4. Try: ha addons reload from the terminal, or restart HA entirely
Add-on starts but the Web UI shows a blank page or connection refused
1. Check the Log tab in the add-on — look for Python errors
2. Verify port 7123 isn't used by another service: netstat -tlnp | grep 7123
3. If behind a proxy/firewall, ensure port 7123 is allowed
4. Try accessing via ingress: http://homeassistant.local:8123/hassio/ingress/apex_trading
Build fails with Python package errors
The Dockerfile installs Python packages during build. Common fixes:

1. Ensure the HA host has internet access during the build
2. If pip3 install fails, the Dockerfile tries both with and without --break-system-packages
3. Check the build log for the specific package that fails
4. If on Raspberry Pi (armhf), builds take longer — wait 5-10 minutes
Broker connection returns "Could not extract balance"
1. Check the Log tab for the actual HTTP error code from your broker
2. Common causes: wrong URL (double-check format in Broker table above), API key doesn't have balance read permission, IP whitelist on broker side blocking HA's IP
3. For Binance: check "Enable Reading" permission on your API key
4. For Kraken: add your HA's external IP to the API key allowlist
5. Use Manual Capital entry as a workaround while debugging
Market prices aren't updating / stuck on old values
1. Click 📡 Fetch Live Prices Now in the sidebar
2. Check Market Data status indicator — if red, there's a fetch error
3. Yahoo Finance occasionally has outages — try switching to Finnhub
4. Check the add-on Log for error messages from the price fetcher
5. Verify the HA host can reach the internet: curl -s https://query1.finance.yahoo.com/v8/finance/chart/SPY?range=1d | head -c 200
Claude AI gives "No Anthropic API key" error
1. Enter your key in the APEX sidebar Claude AI panel (inline field) — this saves to the server immediately
2. Or set it in the add-on Configuration tab and restart the add-on
3. Verify the key is valid: it starts with sk-ant-
4. Check your Anthropic account has credits at console.anthropic.com
How do I view/backup my data?
All data is in /addon_configs/apex_trading/apex/ (accessible via Samba as the addon_configs share):

• state.json — portfolio, prices, strategy weights
• trades.json — full trade history
• analyses.json — all Claude AI analyses
• optlog.json — optimization log

Backup: copy these 4 files. Restore: paste them back and restart the add-on.
How do I update APEX to a new version?
1. Download the new apex-trading-addon.zip
2. Stop the APEX add-on in HA
3. Re-run the installer: ./install.sh (it detects existing installation and backs up data)
4. OR manually copy new files: cp -r apex_trading/rootfs/opt/apex/* /addons/apex_trading/rootfs/opt/apex/
5. Rebuild and restart the add-on in HA

Your data in /addon_configs/ is always preserved.

ARCHITECTURE
How APEX works inside HAOS
┌─────────────────────────────────────────────────────────────┐ │ Home Assistant OS Host │ │ │ │ ┌──────────────────────────────────────────────────────┐ │ │ │ APEX Trading AI Container │ │ │ │ │ │ │ │ ┌──────────────────┐ ┌───────────────────────┐ │ │ │ │ │ aiohttp server │ │ /data/apex/ │ │ │ │ │ │ port 7123 │───▶│ state.json │ │ │ │ │ │ │ │ trades.json │ │ │ │ │ │ /api/broker/* │ │ analyses.json │ │ │ │ │ │ /api/prices/* │ │ optlog.json │ │ │ │ │ │ /api/claude/* │ └───────────────────────┘ │ │ │ │ │ /static/ │ │ │ │ │ └────────┬─────────┘ │ │ │ │ │ │ │ │ │ Server-side outbound calls (no CORS): │ │ │ │ ✓ → Alpaca/Binance/Kraken (broker API) │ │ │ │ ✓ → Yahoo Finance / Finnhub (market data) │ │ │ │ ✓ → api.anthropic.com (Claude AI) │ │ │ │ │ │ │ └──────────────────────────────────────────────────────┘ │ │ │ │ ┌─────────────┐ HA Ingress ┌───────────────────────┐ │ │ │ HA Core │◀─────────────▶│ Your Browser │ │ │ │ port 8123 │ │ APEX Dashboard UI │ │ │ └─────────────┘ │ (fetch → /api/*) │ │ │ └───────────────────────┘ │ └─────────────────────────────────────────────────────────────┘

The browser only ever talks to /api/* on the same HA origin — no external API calls from the browser. All external calls (broker, market data, Claude) are made by the Python server running inside the container, which has no CORS restrictions. Data is stored in JSON files on persistent HA storage, surviving reboots and HA updates.
